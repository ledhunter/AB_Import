# Импорт "Финмодель" - обновление параметров объекта строительства

**Дата**: 2026-04-30  
**Автор**: Cascade AI  
**Тип импорта**: `finmodel`

## Описание

Импорт типа "Финмодель" предназначен для обновления параметров объекта строительства на основе данных из Excel файла. В текущей версии поддерживается обновление поля **"Тип отделки"** (FinishingMaterial).

## Архитектура

### Backend

**Маппер**: `FinModelImportMapper` (`KiloImportService.Api/Domain/Mapping/FinModelImportMapper.cs`)

**Основные компоненты**:
- Валидация данных из Excel
- Маппинг названий типов отделки в ID справочника
- Обновление сущности `ConstructionSite` в базе данных Visary

**Справочник "Тип отделки"**:
| Название | ID |
|----------|-----|
| Черновая | 3 |
| Предчистовая | 2 |
| Чистовая | 1 |

### Frontend

**Компоненты**:
- `ImportForm` - форма выбора проекта и объекта строительства
- `FileUpload` - компонент загрузки Excel файла
- `ImportTypePicker` - выбор типа импорта "Финмодель"

## Формат входного файла

### Структура Excel файла

Файл должен содержать лист с именем `inputs` (или любым другим) со следующими колонками:

| Колонка | Обязательность | Допустимые значения | Примечание |
|---------|----------------|---------------------|------------|
| Тип отделки | Обязательно | Черновая, Предчистовая, Чистовая | Регистр не важен |

**Альтернативные названия колонок**:
- "Тип отделки"
- "FinishingType"
- "Finishing"

### Пример файла

```
| Тип отделки  |
|--------------|
| Черновая     |
```

## Процесс импорта

### 1. Загрузка файла

```typescript
// Frontend
const payload = {
  importTypeCode: 'finmodel',
  file: selectedFile,
  projectId: selectedProjectId,
  siteId: selectedSiteId,
};

await uploadImport(payload);
```

### 2. Валидация

Маппер проверяет:
1. ✅ Наличие объекта строительства (siteId)
2. ✅ Существование объекта в базе данных
3. ✅ Наличие колонки "Тип отделки"
4. ✅ Непустое значение
5. ✅ Соответствие значения справочнику

### 3. Применение

При успешной валидации:
1. Загружается объект `ConstructionSite` из БД
2. Обновляется поле `FinishingMaterialId`
3. Изменения сохраняются в базе данных Visary

## Использование

### Через UI

1. Откройте сервис импорта: `http://localhost:5174`
2. Выберите тип импорта: **"Финмодель"**
3. Выберите проект из списка
4. Выберите объект строительства
5. Загрузите Excel файл с колонкой "Тип отделки"
6. Нажмите **"Запустить импорт"**
7. Дождитесь завершения валидации
8. Нажмите **"Применить"** для сохранения изменений

### Программно (API)

```bash
# 1. Загрузить файл
curl -X POST http://localhost:5000/api/imports \
  -F "importTypeCode=finmodel" \
  -F "file=@finmodel.xlsx" \
  -F "siteId=7847"

# Ответ: { "sessionId": "guid", "status": "Parsing" }

# 2. Дождаться валидации (через SignalR или polling)

# 3. Применить изменения
curl -X POST http://localhost:5000/api/imports/{sessionId}/apply
```

## Обработка ошибок

### Ошибки уровня файла

| Код ошибки | Сообщение | Решение |
|------------|-----------|---------|
| `site_required` | Не указан объект строительства | Выберите объект в UI |
| `site_not_found` | Объект не найден или скрыт | Проверьте ID объекта |

### Ошибки уровня строки

| Код ошибки | Сообщение | Решение |
|------------|-----------|---------|
| `column_not_found` | Не найдена колонка 'Тип отделки' | Добавьте колонку в Excel |
| `value_empty` | Значение пустое | Заполните ячейку |
| `invalid_value` | Неизвестный тип отделки | Используйте: Черновая, Предчистовая, Чистовая |

## Расширение функционала

### Добавление новых параметров

Для добавления поддержки других параметров объекта строительства:

1. Добавьте поле в сущность `ConstructionSite.cs`
2. Добавьте алиасы колонок в `FinModelImportMapper.cs`
3. Обновите логику валидации в `ValidateAsync`
4. Обновите логику применения в `ApplyAsync`
5. Добавьте тесты

### Пример: добавление "Класс энергоэффективности"

```csharp
// 1. Добавить поле
public class ConstructionSite
{
    // ...
    public int? EnergyEfficiencyClassId { get; set; }
}

// 2. Добавить в маппер
private static readonly string[] EnergyClassAliases = 
    ["Класс энергоэффективности", "EnergyClass"];

// 3. Валидация
var energyClassValue = row.Cells[energyClassCol]?.Trim();
var energyClassId = GetEnergyClassId(energyClassValue);

// 4. Применение
site.EnergyEfficiencyClassId = energyClassId;
```

## Известные ограничения

1. ⚠️ Обновляется только один объект строительства за раз
2. ⚠️ Файл должен содержать только одну строку с параметрами
3. ⚠️ Требуется подключение к базе данных Visary
4. ⚠️ Нет поддержки массового обновления нескольких объектов
